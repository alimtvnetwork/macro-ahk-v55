# Chrome Extension — User Script Error Isolation

**Version**: v1.0.0  
**Date**: 2026-02-28  
**Fixes**: Risk R-05 (User Script Error Capture in MAIN World)  
**Replaces**: Error Handling section in `13-script-and-config-management.md`

---

## Purpose

Define how user-uploaded scripts are wrapped with error handling before injection, ensuring:

1. **Isolation**: User script errors are captured without catching unrelated page errors
2. **Attribution**: Every error is tagged with `scriptId`, `projectId`, `configId`
3. **Non-interference**: A failing user script does NOT crash other scripts or the extension
4. **Both worlds**: Works in both ISOLATED and MAIN execution worlds

---

## Problem with Previous Approach

The v0.2 spec (`13-script-and-config-management.md` §Error Handling) used `window.addEventListener('error', ...)` with an `isUserScript(event.filename)` filter. This fails because:

1. `chrome.scripting.executeScript` with `func` parameter shows `<anonymous>` as filename — no way to identify which script
2. `window.onerror` captures ALL page errors in MAIN world, not just ours
3. No `filename` attribute exists when scripts are injected as inline code
4. Race condition: error listener must be injected before user scripts, but injection order isn't guaranteed with parallel `executeScript` calls

---

## Solution: Try/Catch Wrapper at Injection Time

Every user script is wrapped in a try/catch block **before injection**. The wrapper is generated by the background service worker and injected as a single unit.

### Wrapper Template

```typescript
// src/content-scripts/error-wrapper.ts

export function wrapUserScript(
  scriptContent: string,
  context: {
    scriptId: string;
    scriptName: string;
    projectId: string;
    configId: string | null;
    urlRuleId: string;
  }
): string {
  // Escape the context for safe embedding
  const ctx = JSON.stringify(context);

  return `
(function __marcoScriptWrapper() {
  'use strict';
  var __ctx = ${ctx};
  try {
    // ── User Script Start ──
    ${scriptContent}
    // ── User Script End ──
  } catch (__err) {
    // Parse stack trace for line/column (offset by wrapper lines)
    var __line = null, __col = null;
    if (__err.stack) {
      var __match = __err.stack.match(/<anonymous>:(\\d+):(\\d+)/);
      if (__match) {
        __line = parseInt(__match[1], 10) - 5; // Subtract wrapper preamble lines
        __col = parseInt(__match[2], 10);
      }
    }

    // Route error to background via message
    try {
      chrome.runtime.sendMessage({
        type: 'USER_SCRIPT_ERROR',
        scriptId: __ctx.scriptId,
        scriptName: __ctx.scriptName,
        projectId: __ctx.projectId,
        configId: __ctx.configId,
        urlRuleId: __ctx.urlRuleId,
        message: __err.message || String(__err),
        stack: __err.stack || '',
        line: __line,
        column: __col,
        url: window.location.href
      });
    } catch (_sendErr) {
      // If message send fails, at least log to console
      console.error('[Marco] Script error in ' + __ctx.scriptName + ':', __err);
    }
  }
})();
`;
}
```

### Injection with Wrapper

```typescript
// src/background/injector.ts

import { wrapUserScript } from '@/content-scripts/error-wrapper';
import { MessageType } from '@/shared/messages';

async function injectScript(
  tabId: number,
  script: StoredScript,
  binding: ScriptBinding,
  context: { projectId: string; urlRuleId: string; configId: string | null }
): Promise<InjectionResult> {
  const wrappedCode = wrapUserScript(script.content, {
    scriptId: script.id,
    scriptName: script.name,
    projectId: context.projectId,
    configId: context.configId,
    urlRuleId: context.urlRuleId,
  });

  try {
    await chrome.scripting.executeScript({
      target: { tabId },
      world: binding.world,
      func: new Function(wrappedCode) as () => void,
      // Note: Using `func` with dynamically created function.
      // Alternative: Use `args` + a static wrapper function
    });

    return { scriptId: script.id, status: 'success' };
  } catch (injectionErr) {
    return {
      scriptId: script.id,
      status: 'error',
      error: (injectionErr as Error).message,
    };
  }
}
```

**Important**: The `func` approach with `new Function()` may not work in all contexts due to CSP. The fallback uses `chrome.scripting.executeScript` with `args`:

```typescript
// Fallback: inject a static wrapper function with args
async function injectScriptFallback(
  tabId: number,
  wrappedCode: string,
  world: 'ISOLATED' | 'MAIN'
): Promise<void> {
  await chrome.scripting.executeScript({
    target: { tabId },
    world,
    func: (code: string) => {
      // This runs in the target page/isolated world
      const script = document.createElement('script');
      script.textContent = code;
      document.documentElement.appendChild(script);
      script.remove();
    },
    args: [wrappedCode],
  });
}
```

## Async Script Support

Some user scripts use `async/await` at the top level. The wrapper supports this:

```typescript
export function wrapUserScriptAsync(
  scriptContent: string,
  context: ScriptContext
): string {
  const ctx = JSON.stringify(context);

  return `
(async function __marcoScriptWrapperAsync() {
  'use strict';
  var __ctx = ${ctx};
  try {
    ${scriptContent}
  } catch (__err) {
    // Same error routing as sync version
    try {
      chrome.runtime.sendMessage({
        type: 'USER_SCRIPT_ERROR',
        scriptId: __ctx.scriptId,
        scriptName: __ctx.scriptName,
        projectId: __ctx.projectId,
        configId: __ctx.configId,
        urlRuleId: __ctx.urlRuleId,
        message: __err.message || String(__err),
        stack: __err.stack || '',
        url: window.location.href
      });
    } catch (_) {
      console.error('[Marco] Script error in ' + __ctx.scriptName + ':', __err);
    }
  }
})();
`;
}
```

### Auto-Detection

The wrapper generator checks if the script content contains top-level `await`:

```typescript
export function wrapScript(content: string, context: ScriptContext): string {
  const hasTopLevelAwait = /\bawait\s/.test(content) && !/function\s/.test(content.split('await')[0]);
  // Simplified heuristic — can be improved with AST analysis

  if (hasTopLevelAwait || content.includes('async ')) {
    return wrapUserScriptAsync(content, context);
  }
  return wrapUserScript(content, context);
}
```

## Unhandled Promise Rejections (MAIN World Only)

For MAIN world scripts, unhandled promise rejections won't be caught by try/catch. Add a scoped listener:

```typescript
// Added inside the wrapper for MAIN world scripts
window.addEventListener('unhandledrejection', function __marcoRejectionHandler(event) {
  // Only handle if the rejection came from our script's execution context
  // We can't perfectly filter, but we can check timing
  chrome.runtime.sendMessage({
    type: 'USER_SCRIPT_ERROR',
    scriptId: __ctx.scriptId,
    scriptName: __ctx.scriptName,
    projectId: __ctx.projectId,
    message: 'Unhandled Promise rejection: ' + (event.reason?.message || String(event.reason)),
    stack: event.reason?.stack || '',
    url: window.location.href
  });
}, { once: true }); // Only catch the first one to avoid noise
```

## Line Number Correction

Stack traces from wrapped scripts have line numbers offset by the wrapper preamble. The wrapper subtracts the known offset:

| Wrapper Type | Preamble Lines | Offset |
|-------------|---------------|--------|
| Sync (`wrapUserScript`) | 5 lines before user code | -5 |
| Async (`wrapUserScriptAsync`) | 5 lines before user code | -5 |

The corrected line number is sent in the `USER_SCRIPT_ERROR` message and stored in `errors.db` as `error_line`.

## Config Injection Method 3 — Removed

**Previous spec** defined Method 3 (Parameter Wrapping) which wrapped user scripts in `(function(config) { ... })({config})`. This is **removed** because:

1. Breaks scripts with top-level `return` statements
2. Breaks scripts with top-level `await`
3. Breaks scripts with `import` statements
4. Conflicts with the try/catch wrapper (double-wrapping)

**Only two config injection methods remain**:

| Method | Name | World | Description |
|--------|------|-------|-------------|
| 1 | Global Variable | MAIN only | `window.__marcoConfig = {config}` injected before script |
| 2 | Message Passing | Both | Script calls `chrome.runtime.sendMessage({ type: 'GET_SCRIPT_CONFIG' })` |

Method 2 is the default for all new scripts. See `12-project-model-and-url-rules.md` for details.
