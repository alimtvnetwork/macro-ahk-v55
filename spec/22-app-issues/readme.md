# App Issues

## Overview

Tracked app-level issues and their resolution specs. Each numbered file is one issue (workspace name mis-display, status-bar credit mismatch, 405 API failures, DOM-observer overwrite races, etc.). Files document repro, root-cause analysis, fix design, and acceptance criteria — they are the durable record of why a given fix exists. Open issues live in `plan.md` (root); closed issues are archived here for future LLM context.

When fixing an issue, append the resolution + memory anchor to the matching file rather than deleting it.

- [`00-overview.md`](./00-overview.md) — issue index
- Numbered issue files `01-…` through `128-…` (one per tracked issue) — see directory listing
