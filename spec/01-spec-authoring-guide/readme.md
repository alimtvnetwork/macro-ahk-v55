# Spec Authoring Guide

## Overview

How to author new spec files. Defines folder structure (`01`), naming conventions (`02`), required files per spec dir (`03` — H1, Overview, Files; enforced by `scripts/check-spec-readme-structure.mjs`), the AI-onboarding prompt (`04`), spec templates for CLI/app/non-CLI modules (`04`–`06`), the `memory/` folder guide (`07`), cross-references (`08`), and the exceptions list (`09-exceptions.md` — mirrors the `readme.txt` hard ban in `mem://constraints/readme-txt-prohibitions`).

Use this directory before creating any new `spec/*/` folder.
