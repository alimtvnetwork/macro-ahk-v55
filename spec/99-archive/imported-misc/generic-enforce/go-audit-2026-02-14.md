## Category 3: `map[string]any` inline literals (MEDIUM — scattered throughout services)

These are `map[string]any{...}` constructed at call sites, passed to broadcast/log functions.

### ws/hub.go broadcast methods (~12 methods)

All convenience broadcast methods accept `details map[string]any`:
- `BroadcastError(code, message string, context map[string]any)`
- `BroadcastConnectionTestProgress(..., details map[string]any)`
- `BroadcastLog(level, message string, context map[string]any)`
- `BroadcastPublishLog(..., details map[string]any)`
- `BroadcastSyncLog(..., details map[string]any)`
- `BroadcastBackupLog(..., details map[string]any)`
- `BroadcastRemotePluginLog(..., details map[string]any)`

**Fix**: Each broadcast method should accept its own typed details struct. Example: `BroadcastPublishLog(..., details PublishLogDetails)`.

### Service call sites (~200+ inline literals)

Files with highest `map[string]any{...}` literal density:
1. `services/site/service.go` — ~30 literals (connection test progress, deploy progress)
2. `services/backup/service.go` — ~15 literals (backup CRUD progress)
3. `services/publish/scheduler.go` — ~20 literals (publish progress)
4. `services/sync/service.go` — ~15 literals (sync progress)
5. `services/session/service.go` — ~10 literals (session logging)
6. `services/e2e/service.go` — ~10 literals (test execution)

**Fix**: Define typed structs per operation domain. Each literal becomes a struct constructor.

## Remediation Priority

### Phase 1: Data model types (blocks everything else)
1. Replace `apperror.ErrorContext` alias with concrete struct
2. Replace all `map[string]any` in ws broadcast data structs with typed structs
3. Replace all `any` fields in `response_types.go` with concrete types
4. Replace `dbops.ContextFields` alias with typed struct
5. Define typed details structs for each operation domain (publish, sync, backup, site, session)

### Phase 2: Framework generics
6. Make `envelope.Success/Created/List/ListUnpaginated` generic `[T any]`
7. Make `ws.Hub.Broadcast/BroadcastWithSession` generic `[T any]`
8. Make `handlers.respondSuccess/respondCreated/respondList` generic `[T any]`
9. Make `handlers.requireService/decodeJSON` generic `[T any]`
10. Make handler factory functions generic

### Phase 3: Business signatures
11. Update all service interfaces to use typed params (no `map[string]any`)
12. Update all broadcast call sites to construct typed structs instead of map literals
13. Update `apperror.WithContext` to accept typed struct

### Phase 4: Service lazy getters
14. Replace `siteService() any` etc. with concrete return types (`site.ServiceInterface`)
