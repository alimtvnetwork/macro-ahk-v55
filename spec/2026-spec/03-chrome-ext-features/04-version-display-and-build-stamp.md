# 04 — Version Display and Build Stamp

## Why this step exists

When a user files a bug ("the popup is blank"), the first question is:
**which build?** Without a visible, copy-pasteable version + build id surfaced
in every active UI surface, runtime log, and diagnostics export, the developer
wastes hours guessing. This step pins the contract so the same build identity
appears everywhere and cannot drift.

## Hard boundary: no README timestamps

`BUILD_TIME_ISO`, diagnostics `manifest.txt`, and `EXPORTED_AT` are allowed
only in generated source metadata or diagnostics artifacts. They MUST NOT be
written to `readme.txt`, proposed for `readme.txt`, formatted for `readme.txt`,
or used by README timestamp automation.

## Contract

1. **Single source of truth**: `manifest.json#version` is the canonical semver.
   `package.json#version` is a manually maintained mirror. The prebuild
   verifier fails if it differs; it MUST NOT silently rewrite `package.json`.
2. **Generated version module**: a build script reads the canonical version and
   emits only `src/shared/generated/version.ts`:
   ```ts
   // AUTO-GENERATED by scripts/stamp-version.mjs — do not edit.
   export const VERSION = "3.40.0";
   export const BUILD_ID = "3.40.0+ab12cd3";
   export const BUILD_TIME_ISO = "2026-06-05T03:14:00.000Z";
   ```
   The generator MUST NOT overwrite `src/shared/constants.ts` or any non-version
   constants (`ID_*`, `SEL_*`, `ATTR_*`, `CSS_*`, `MSG_*`, `EVT_*`).
3. **Mirror availability**: runtime code imports from
   `@shared/generated/version`. The service worker never reads `manifest.json`
   at runtime because MV3 has no filesystem.
4. **Pre-build verification** fails fast if `manifest.json#version`,
   `package.json#version`, and `src/shared/generated/version.ts#VERSION`
   disagree.
5. **UI surfaces**:
   - Current required surface: popup footer (always visible).
   - Required once implemented: options page header.
   - Required once implemented: in-page floating panel header.
   - A surface that does not exist MUST NOT be created solely to show the build
     id; it becomes obligated when the surface exists for its own step.
6. **Diagnostics and logs**:
   - Diagnostics ZIP filename and root `manifest.txt` include `BUILD_ID`.
   - Every Code Red error log row includes the canonical build field from step
     11/12.
7. **Click-to-copy**: clicking the visible `BUILD_ID` label attempts to copy the
   raw string. Copy failure must not crash the surface.
8. **Reload refresh** (see step `05-extension-reload-manual.md`):
   - Popup/options: next open/render imports the new bundled version module.
   - Background logs: the restarted worker writes the new `BUILD_ID`.
   - Existing in-page panel: shows the new `BUILD_ID` only after a version
     refresh message, reinjection, or page reload; no stronger guarantee is
     implied.

## Build-id resolution

`BUILD_ID` has the format `<semver>+<short-sha-or-descriptor>`.

Resolution precedence is deterministic:

1. `process.env.BUILD_SHA` when set; shorten to 7–12 hexadecimal characters.
2. `git rev-parse --short HEAD` when `.git` is available.
3. Release descriptor / `.gitmap` value if this repo ships without `.git`.
4. `nogit` only for local development.

Release packaging MUST fail if the resolved suffix is `nogit`. Dirty-tree or
detached-HEAD states MAY still produce a build id, but release scripts SHOULD
also enforce their own clean-tree policy outside this step.

## Build-time generation mode

The repo uses **environment-stable generated module mode**:

- `BUILD_TIME_ISO` is UTC ISO-8601.
- If `SOURCE_DATE_EPOCH` is set, use that epoch seconds value.
- Else if `BUILD_TIME_ISO` is set in the environment, use it after ISO parsing.
- Else local development may use `new Date().toISOString()`.
- CI/release builds SHOULD pass one of the stable environment values to avoid
  accidental snapshot churn.

`src/shared/generated/version.ts` may be committed, but the generator must be
idempotent when the inputs are unchanged.

## Reference generator requirements

`scripts/stamp-version.mjs` runs before build and MUST:

1. Read `manifest.json` and `package.json`.
2. Fail if versions differ, without rewriting `package.json`.
3. Resolve the build suffix using §"Build-id resolution".
4. Resolve build time using §"Build-time generation mode".
5. Write only `src/shared/generated/version.ts`.
6. Preserve every existing non-version file and constant.
7. Emit Code-Red-formatted **build errors** through a shared formatter, not
   runtime namespace logging:

```js
formatCodeRedBuildError({
  Path: "manifest.json",
  Missing: "matching package.json version",
  Reason: "VersionMismatch",
  ReasonDetail: "manifest=3.40.0 package=3.39.0",
});
```

Build-script Code Red formatting MUST include `Path`, `Missing`, `Reason`, and
`ReasonDetail`. Runtime Code Red payload requirements remain defined by steps
11 and 12.

## Reference UI behavior

The version badge may be implemented in the existing UI framework or plain DOM.
It MUST render a visible text node and a stable test attribute:

```ts
import { BUILD_ID } from "@shared/generated/version";

export function renderVersionBadge(button: HTMLButtonElement): void {
  button.type = "button";
  button.dataset.buildId = BUILD_ID;
  button.textContent = `v${BUILD_ID}`;
  button.title = "Copy build id";
  button.addEventListener("click", () => {
    copyBuildIdSafely(BUILD_ID, button);
  });
}
```

Copy contract:

- Try `navigator.clipboard.writeText(BUILD_ID)` only inside the user gesture.
- If clipboard is unavailable or rejects, fall back to selecting a hidden text
  node or mark `data-copy-state="failed"` without throwing.
- Clipboard denial is a warning at most, never Code Red.

The `data-build-id` attribute lets E2E tests assert the visible value without
parsing arbitrary text.

## Runtime Code Red integration

Every Code Red runtime log row written via the namespace logger MUST carry the
canonical build field and the mandatory diagnostic fields from steps 11 and 12:

```ts
Logger.error("Injection.Failed", {
  BuildId: BUILD_ID,
  Path: "src/injected/sdk.ts",
  Missing: "window.RiseupAsiaMacroExt symbol",
  Reason: "MainWorldGlobalMissing",
  ReasonDetail: "MAIN-world script returned before stamping global.",
  SelectorAttempts: null,
  VariableContext: null,
});
```

If a later step defines the final casing as `buildId`, use the canonical step
11/12 casing consistently everywhere; do not mix casing in one table.

## Diagnostics ZIP integration

The exported diagnostics bundle must:

- Name the file with sanitized values:
  ```text
  fileSafeBuildId = BUILD_ID.replace(/[^a-zA-Z0-9._-]/g, "_")
  fileSafeIso = exportedAtIso.replace(/[:]/g, "-")
  diagnostics-${fileSafeBuildId}-${fileSafeIso}.zip
  ```
- Keep raw values inside root `manifest.txt`:
  ```text
  BUILD_ID=3.40.0+ab12cd3
  BUILD_TIME=2026-06-05T03:14:00.000Z
  USER_AGENT=Mozilla/5.0 …
  EXPORTED_AT=2026-06-05T07:45:11.000Z
  ```

The raw `manifest.txt` is a diagnostics artifact, not a README or changelog.

## Pitfalls

- **Overwriting `src/shared/constants.ts`** — deletes unrelated constants and
  breaks consumers. Generate `src/shared/generated/version.ts` only.
- **Copying build timestamps into `readme.txt`** — strictly forbidden.
- **Reading `chrome.runtime.getManifest().version` everywhere** — no git/hash
  suffix. Use `BUILD_ID` from `@shared/generated/version`.
- **Silently rewriting `package.json#version`** — hides drift. Fail instead.
- **Allowing `nogit` in release output** — local-dev fallback only.
- **Showing `BUILD_ID` only on hover** — invisible to screenshot bug reports.
- **Crashing on clipboard failure** — copy is convenience behavior, not a fatal
  path.
- **Using raw ISO strings in ZIP filenames** — `:` is invalid or awkward on
  Windows paths; sanitize filenames and keep raw values inside the file.

## Acceptance

A reviewer can answer "yes" to every line:

- [ ] `manifest.json#version` is canonical; `package.json#version` is verified
      but never silently rewritten.
- [ ] `src/shared/generated/version.ts` is the only generated version output;
      `src/shared/constants.ts` and non-version constants are preserved.
- [ ] `manifest.json`, `package.json`, and generated `VERSION` carry the
      identical semver after a fresh stamp.
- [ ] `BUILD_ID` uses deterministic suffix precedence and release packaging
      rejects `nogit`.
- [ ] `BUILD_TIME_ISO` is UTC ISO and respects `SOURCE_DATE_EPOCH` or
      environment-stable inputs when present.
- [ ] No generated timestamp or git-update text is written to `readme.txt` or
      proposed for `readme.txt` automation.
- [ ] `BUILD_ID` is visible, not hover-only, in every currently implemented
      user-visible extension surface; popup is mandatory now, options/panel are
      mandatory once those surfaces exist.
- [ ] Clicking the version badge copies `BUILD_ID` or records a non-crashing
      copy-failed state.
- [ ] Every runtime Code Red log row contains the canonical build field plus
      `Reason`, `ReasonDetail`, `SelectorAttempts`, and `VariableContext`.
- [ ] Diagnostics ZIP filename uses file-safe values and root `manifest.txt`
      contains raw `BUILD_ID` and build/export metadata.
- [ ] Reload behavior is surface-scoped: popup/options next render, background
      next worker, panel only after message/reinject/page reload.
- [ ] Pre-build failures are Code-Red-formatted build errors with exact path,
      missing item, `Reason`, and `ReasonDetail`.

## Tests to ship with this step

- `scripts/__tests__/stamp-version.test.mjs` — runs the generator against
  fixture manifests, asserts generated `src/shared/generated/version.ts`.
- `scripts/__tests__/version-sync.test.mjs` — reads `manifest.json`,
  `package.json`, and generated `VERSION`, then asserts equality.
- `scripts/__tests__/version-preserves-constants.test.mjs` — proves stamping
  does not overwrite `src/shared/constants.ts` or non-version constants.
- `scripts/__tests__/version-release-sha.test.mjs` — proves release mode fails
  when SHA resolves to `nogit` and accepts `BUILD_SHA` / git / descriptor
  sources in precedence order.
- `scripts/__tests__/build-time-stability.test.mjs` — proves
  `SOURCE_DATE_EPOCH` and `BUILD_TIME_ISO` inputs generate stable UTC ISO.
- `scripts/__tests__/readme-timestamp-boundary.test.mjs` — fails if the version
  stamping scripts write or propose timestamp/git-update values for
  `readme.txt`.
- Popup/component test — renders the version badge, asserts `data-build-id`
  matches `BUILD_ID`, asserts copy success and copy failure do not crash.
- Logger payload validator test — rejects Code Red rows missing the canonical
  build field, `Reason`, `ReasonDetail`, `SelectorAttempts`, or
  `VariableContext`.
- Diagnostics export test — asserts file-safe ZIP name and raw values inside
  root `manifest.txt`.

## Determinism (MUST)

- **MUST** bind every extension numeric (alarm intervals, debounce ms, retry counts=0, sentinel TTL, badge text limits) to a constant in `src/shared/constants.ts` or a local `*-defaults.ts` module. Inline literals are rejected by code review.
- **MUST** gate auto-injector and project-matcher with `isNewTabOrBlankUrl()` from `src/shared/url-utils.ts` — never run on `about:blank`, `chrome://newtab/`, or empty URLs (see `mem://features/new-tab-no-url-guard`).
- **MUST** route every failure through `RiseupAsiaMacroExt.Logger.error` with `Reason`+`ReasonDetail` and surface boot-time failures via `BootFailureBanner`. Bare `console.error` is rejected by `public/logger-compliance-audit.json`.
- **MUST** pair every `setInterval` / `setTimeout` / `MutationObserver` / event listener with a teardown registered on `pagehide` (see `mem://standards/timer-and-observer-teardown`). Tick UIs MUST pause on `document.hidden`.

## Pitfalls / Counter-examples

- ❌ `catch (caught) { /* ignore */ }` around `chrome.runtime.sendMessage`. ✅ `Logger.error('scope', 'send failed', caught)` and re-throw (see `public/error-swallow-audit.json`).
- ❌ Calling `chrome.scripting.executeScript` on a new-tab URL because the matcher did not gate it. ✅ Always call `isNewTabOrBlankUrl(tab.url)` first; treat true as a non-error skip.
- ❌ Storing a timestamp as `new Date().toLocaleString('en-US', { timeZone: 'Asia/Kuala_Lumpur' })`. ✅ Store `Date.now()` ms UTC; render with `Intl.DateTimeFormat().resolvedOptions().timeZone` (see `mem://localization/timezone`).
- ❌ Retrying `fetch` with `for (let i=0;i<3;i++)` and exponential backoff after a 4xx/5xx. ✅ Use `httpFetchOrThrow` / `httpFailFast` from `src/shared/http-fail-fast.ts`; one attempt, then halt (see `.lovable/checklists/http-fail-fast.md`).
- ❌ Injecting the same content-script twice because the sentinel check was skipped. ✅ Read `#marco-css-sentinel` / data-attribute sentinel before re-injection (see `09-injection-idempotency-sentinel.md`).
