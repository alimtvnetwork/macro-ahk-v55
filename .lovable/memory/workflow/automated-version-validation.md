The build pipeline enforces project integrity through mandatory pre-build validation scripts and compiler checks. 'check-version-sync.mjs' ensures version consistency across six critical files (manifest.json, shared-state.ts, instruction source/dist, seed-manifest.json, and constants.ts). 'check-standalone-dist.mjs' verifies the existence of required artifacts like 'instruction.json'. 'check-axios-version.mjs' validates dependency safety. Additionally, the macro-controller build requires a clean TypeScript compilation ('tsc --noEmit -p tsconfig.macro.build.json'); the pipeline is configured to break if any TypeScript errors or specific lint violations are detected. To ensure validators see fresh data, 'build:seed-manifest' and 'compile-instruction.mjs' are executed before these checks in the pipeline sequence.
