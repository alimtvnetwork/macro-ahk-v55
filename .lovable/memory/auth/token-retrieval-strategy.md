To eliminate 401 errors, the system utilizes a zero-network authentication resolution waterfall where JWTs are resolved strictly from direct cookies, platform localStorage (including all subframes via 'allFrames: true'), or signed URL tokens ('__lovable_token') found across any active frame via 'webNavigation.getAllFrames'. The legacy '/auth-token' network fallback is entirely removed. Furthermore, the 'marco-sdk' uses a robust normalization utility ('auth-response.ts') to extract raw JWT strings from bridge payloads, preventing 'Bearer [object Object]' header corruption caused by inconsistent object wrappers in the bridge response.
