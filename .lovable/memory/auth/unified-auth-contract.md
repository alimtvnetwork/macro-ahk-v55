The authentication system follows a unified single-path contract managed by the 'getBearerToken()' and 'getBearerToken({ force: true })' methods. All operational paths—including startup, workspace moves, and API renaming—are required to use this contract to ensure deterministic behavior. Legacy functions ('resolveToken', 'recoverAuthOnce', 'invalidateSessionBridgeKey') have been completely removed to prevent competing auth states. The system prioritizes a fast load by minimizing redundant token resolution calls and aligning timeouts with the background bridge's performance. All components reading or writing tokens must interact with the shared 'localStorage' source managed by this bridge.
