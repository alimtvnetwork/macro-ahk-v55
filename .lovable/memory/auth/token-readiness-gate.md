The startup token gate (v2.139.0) uses a unified 10s readiness budget shared across startup, bridge recovery (AuthRecoveryManager), and background (config-auth-handler). Before entering the async waterfall, a synchronous fast pre-seed phase extracts JWTs from signed URL params (__lovable_token) directly into marco_bearer_token, enabling sub-2s cold-load resolution. The gate delegates to getBearerToken() for the async path, which uses the same ConcurrencyLock timeout. The StartupGateSnapshot includes preSeedSource for diagnostics.
