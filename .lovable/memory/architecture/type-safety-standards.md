For the macro-controller build, global types must be declared within a 'declare global {}' block in 'globals.d.ts' to be recognized in the module-based environment. When casting strongly-typed objects (like MacroControllerConfig or ControllerState) to 'Record<string, unknown>', they must first be cast to 'unknown' to satisfy TypeScript's overlap requirements. These standards were finalized during the v2.01.0 build stabilization.
