Error state changes are broadcast across extension contexts using an 'ERROR_COUNT_CHANGED' message pattern to ensure UI synchronization. When the background service worker detects a change—either a new error via the bridge or a manual clearing of errors—it recalculates the unresolved count from SQLite and broadcasts it to all active tabs. This allows UI elements like the sidebar notification badge to update instantly without polling. The 'useErrorCount' hook maintains a secondary slow-polling fallback to ensure eventual consistency in disconnected or non-extension environments.
