The system implements a robust two-stage self-healing mechanism via 'builtin-script-guard.ts' to ensure storage integrity for core scripts (macro-looping.js, marco-sdk.js, xpath.js). If these scripts are missing from 'chrome.storage.local', the system recursively resolves transitive dependencies and first attempts an automatic reseed from the primary manifest ('seedFromManifest'). If that returns zero results, it executes a secondary fallback ('seedMissingBuiltinsDirectly') which seeds scripts directly from their individual 'instruction.json' metadata and 'web_accessible_resources' using a hardcoded 'BUILTIN_DIST_MAP'. This dual-path guard is enforced across all injection entry points (manual popup, auto-injector, and SPA re-injection) to prevent '0 injected' failures.
