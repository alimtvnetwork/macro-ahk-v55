The project uses a declarative seeding architecture where 'instruction.ts' is the sole manifest source. A build-generated 'seed-manifest.json' drives the 'manifest-seeder.ts' at runtime. The build pipeline (via 'compile-instruction.mjs') evaluates instruction files using a preamble that captures all top-level constant declarations (e.g., 'LOVABLE_BASE_URL') before the instruction block to prevent evaluation failures. The system enforces 'schemaVersion' validation: versions exceeding the supported maximum trigger a hard rejection, while legacy versions issue a warning but proceed with a best-effort update. Project assets are dynamically discovered via the instruction's 'assets' block.
