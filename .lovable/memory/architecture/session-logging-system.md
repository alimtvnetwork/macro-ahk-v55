The system employs a dual-layered logging strategy where events are written to both SQLite and the Origin Private File System (OPFS). OPFS logs are organized by session ID (e.g., 'session-logs/session-{id}/') into 'events.log', 'errors.log', and 'scripts.log'. This file-based approach provides a robust audit trail for every session, enabling detailed 'Copy Report' and 'Export ZIP' diagnostic features. To prevent storage bloat, the system automatically prunes session log directories older than 7 days during every new session initialization.
