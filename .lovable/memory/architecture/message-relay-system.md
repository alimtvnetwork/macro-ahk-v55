The system implements a 3-tier message relay architecture for extension-to-page communication: 1) Page scripts (MAIN world) use 'window.postMessage' with a unique 'CorrelationId' to request data, 2) A content script (ISOLATED world) acts as a transparent relay that forwards these messages to the background via 'chrome.runtime', and 3) The background service worker processes the requests (e.g., database queries, token retrieval) and returns responses through the same relay path. This ensures security by isolating page scripts from direct extension API access while allowing robust communication.
