# Memory: architecture/storage/database-id-convention
Updated: 2026-03-21

All SQLite primary key `id` columns strictly use `INTEGER PRIMARY KEY AUTOINCREMENT`. TEXT/GUID/UUID identifiers are never used as primary keys. IDs are auto-generated by SQLite — code must NOT supply `id` values on INSERT, and should use `SELECT last_insert_rowid()` to retrieve the generated ID afterward. When a human-readable semantic identifier is needed (e.g., for prompt seeding deduplication), a separate `slug TEXT UNIQUE` column is used instead of the PK. In TypeScript, IDs are represented as `string` (stringified integers) for message-passing compatibility. This convention is enforced in `src/background/db-schemas.ts` and documented in `spec/08-coding-guidelines/coding-guidelines/database-id-convention.md`.
