The system supports a dynamic script loading mechanism using the 'await RiseupAsiaMacroExt.require("Project.Script")' API. The background service worker acts as the gatekeeper, verifying that the requester has the 'allowDynamicRequests' flag enabled and that the target script is either 'isGlobal' or part of an authorized project. Control for these flags is provided via UI toggles in the project settings panel. To maintain an audit trail, all dynamic loading events—including successful injections, permission denials, and errors—are logged to a dedicated SQLite 'logs.db'.
