The system's full lifecycle is divided into 6 distinct phases: 1) Install + Bootstrap (manifest loading and SQLite init), 2) Tab Navigation (URL matching and dependency sorting), 3) Auth + Token Seeding (bearer token resolution via authBridge), 4) Script Injection (IIFE wrapping and MAIN world execution), 5) Runtime Execution (SDK usage and dynamic require() calls), and 6) User Interaction (popup and options page workflows). This lifecycle is audited via SQLite logging to maintain a complete history of system events from installation to runtime.
