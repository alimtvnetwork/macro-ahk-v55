The script injection pipeline follows a 7-stage lifecycle: 1) Dependency resolution, 2) Namespace bootstrapping, 3) Relay installation, 4) IIFE execution, 5) Script-to-script communication setup, 6) CSP fallback handling, and 7) Dynamic Loading (Runtime). Scripts are bundled as self-contained IIFEs to avoid scope pollution and are injected into the 'MAIN' world via the background service worker (script-resolver.ts) using 'chrome.scripting.executeScript'. A shared global namespace (window.RiseupAsiaMacroExt ??= {}) facilitates inter-script coordination; once initialized, subsequent code references should omit the 'window.' prefix (e.g., using 'RiseupAsiaMacroExt' directly).
