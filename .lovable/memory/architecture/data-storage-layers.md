The system utilizes a four-tier storage architecture: extension-side SQLite (Unlimited, persistent through cleanup), IndexedDB (GBs, removed by browser cleanup), localStorage (5MB, removed by browser cleanup), and chrome.storage.local (10MB/Unlimited, persistent through cleanup). The extension-side SQLite layer (ExtensionDB) requires 'sql-wasm.wasm' to be served from the extension's 'dist/wasm/' directory for initialization (via ensureSqlJs()); initialization failures are reported via a 'BootFailureBanner' in the popup with detailed diagnostic paths and path-based error messages. Logs must not be stored in localStorage. A 'Deployments' table in SQLite (tracking 'version', 'deployedAt', 'isComplete') triggers a localStorage purge when a new version is detected.
