The extension implements build-aware cache invalidation and canonical script prioritization to prevent runtime drift. During boot, the system compares a stored 'STORAGE_KEY_LAST_BUILD_ID' against the current 'build-meta.json' ID and clears the injection cache on mismatch. To ensure the latest code is always used, the seeder replaces stored script code with a 'STUB_PREFIX', forcing the 'script-resolver.ts' to prioritize canonical distribution files mapped in 'BUILTIN_BUNDLED_PATHS' over legacy embedded database code.
