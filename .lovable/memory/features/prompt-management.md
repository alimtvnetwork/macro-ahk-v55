The prompt management system uses a manual-load model with a dual-cache in IndexedDB ('marco_prompts_cache'): 'JsonCopy' (raw array) and 'HtmlCopy' (pre-rendered snapshot). The Macro Controller prioritizes 'HtmlCopy' to eliminate redundant rendering loops, while other consumers use 'JsonCopy'. Loading follows a 3-tier strategy: 1) SDK cache, 2) Extension messaging ('GET_PROMPTS') reading from the 'PromptsDetails' SQLite view, and 3) Hardcoded fallbacks. The cache is 'cache-until-invalidated' (save/delete only) and can be manually refreshed via a 'Load' button in the UI.
