# Memory: features/options-ui/project-detail-layout-standards
Updated: 2026-03-26

The Project Detail View (Options UI) includes a 'General' tab as the primary entry point for project metadata. This tab is organized into five sections: Project Identity (Name, Version, Slug, and PascalCase CodeName), Flags (isGlobal, isRemovable, onlyRunAsDependency), Dependencies (global projects as auto-injected implicit deps + explicit dependencies with resolved/missing status + injection order explanation), Settings (isolation, log level, retry, XPath), and Developer Tooling (LLM Guide + .d.ts downloads). The 'Slug' and 'CodeName' are positioned in a distinct row below the project title. Less frequent configuration tabs (Files, Timing, Auth, Network, Diagnostics) are moved into a 'More' (hamburger) overflow dropdown to maintain a clean interface. UI components utilize defensive rendering with nullish coalescing (?? []) for all .map() operations on project arrays to prevent runtime crashes. The `allProjects` prop is passed from Options.tsx to enable cross-project dependency resolution and global project listing.
