# Memory: features/macro-controller/free-credit-api-detection
Updated: 2026-03-30

Free credit detection uses API-primary approach via `GET /workspaces/{workspaceId}/credit-balance` on a 100s interval, with XPath progress-bar fallback. Implementation lives in `standalone-scripts/macro-controller/src/credit-balance.ts`. Key logic: `hasFreeCredit = daily_remaining >= MinDailyCredit` (configurable, default 2). When below threshold and loop is running, auto-moves to the next workspace following the loop direction (up/down) via `performDirectMove()`. Workspace ID is resolved via `GET /projects/{projectId}/workspace` → `workspace.id` and cached. Config reads from `window.__MARCO_CONFIG__.creditStatus.balance` with keys: `checkIntervalSeconds` (100), `minDailyCredit` (2), `enableApiDetection` (true), `fallbackToXPath` (true). The loop-cycle.ts `runCycle()` calls `checkAndActOnCreditBalance()` first; on failure, falls back to the full `/user/workspaces` flow which also uses the configurable threshold. CreditManager exposes `fetchBalance(workspaceId?)` method. Types: `CreditBalanceResponse`, `CreditBalanceConfig` in credit-types.ts, `CreditBalanceConfigInput` in config-types.ts. Spec: `spec/22-app-issues/free-credits-detect/overview.md`. Status: Implemented v1.74.0.
