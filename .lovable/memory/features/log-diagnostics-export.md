The extension supports a human-readable diagnostics export via a ZIP bundle. This bundle includes a 'logs.txt' file that consolidates database logs and injection events with a summary header tracking key metrics (injected, skipped, and error counts). The export features a per-script breakdown showing the final status (✅ injected, ⏭ skipped, 🔄 guard/reseed, or ❌ error) for each script file. The formatting logic handles both lowercase and PascalCase field names to ensure compatibility with raw SQLite results. The underlying 'DiagnosticDump' and 'ExtensionResponse' types include comprehensive fields for 'wsCount', 'projectId', 'bundledVersion', and 'scriptSource' to provide a complete audit trail for the build and injection environment.
