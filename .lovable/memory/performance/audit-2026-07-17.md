## New findings

| ID | Sev | File:Line | Failure mode | Fix sketch | Expected signal after fix |

| PERF-16 | **HIGH** | `standalone-scripts/macro-controller/src/ui/macro-ui.ts:200,203` | Two 1 Hz `setInterval`s installed per macro-panel mount. No handle stored, no `clearInterval`, no `document.body.contains(...)` bail, no visibility pause. If the panel is rebuilt (redock, workspace switch, hot-reload of injected UI) each mount leaks another pair of ticks that keep running for the tab's lifetime. Second interval also does two `querySelector` + inline-style writes per tick. | Store both handles on the section root (or `interval-registry.ts`); tear down in a `pagehide` + `MutationObserver` "section removed" listener; early-return when `document.hidden`; drive the pause-button state from the `TaskQueueManager` event bus instead of polling. | `interval-registry` size stays flat across remount; no `[macro-ui] tick` while tab hidden. |
| PERF-17 | **HIGH** | `standalone-scripts/macro-controller/src/ui/prompt-utils.ts:493` | 3 s `setInterval` that runs `findPasteTarget()` (XPath walk over the SPA DOM) forever. No teardown, no visibility pause, no bail when the injection host is gone. Every host tab pays this cost even when idle in the background. | Replace with a scoped `MutationObserver(childList+subtree)` on the smallest ancestor of the paste target; disconnect on `pagehide` and when target found+captured; add `document.hidden` early return as a belt-and-suspenders. | Idle-tab CPU drops; `findPasteTarget` calls only fire on real DOM churn. |

## Priority

1. **PERF-16** (macro-ui double interval leak) — highest risk, mounted on every panel and one of the two intervals runs `querySelector` every second.
2. **PERF-17** (prompt-utils 3 s XPath poll) — runs on every host tab that has the extension injected.
3. **PERF-18** (Options page seeder panels) — user-visible tab-idle drain.
4. PERF-19, PERF-20 — small wins, cluster into a single "1 Hz tickers → visibility-paused" sweep.
